用智谱 GLM-4.7 + Composio 打造本地化 Cowork 开发环境

1. 前言：为什么要做这次“心脏移植”？

大家都知道，Anthropic 的 Claude 3.5 Sonnet 虽然强，但 API 调用成本和网络门槛对国内开发者来说始终是个痛点。

最近我在玩 open-claude-cowork 这个开源项目，它提供了一个非常棒的 Electron 客户端，能让你像用 IDE 一样开发 AI Agent。但是，它默认绑定了 Claude 的 API。

这就引出了今天的实战目标：

能不能把它的“心脏”（Claude API）换成国产的 智谱 GLM-4.7，同时保留它强大的 Composio 工具集成能力？

答案是：必须能！ 而且改完之后：

  • 🚀 速度起飞：本地直连，低延迟。
  • 💰 成本打骨折：智谱 API 的价格你懂的，真香。
  • 🛠️ 工具依然强：依然可以调用 GitHub、Gmail 等 500+ 工具。

今天就带大家手把手完成这次“本地化手术”。

2. 准备工作

在动手术前，我们需要准备好手术台和工具。

2.1 必需软件

没啥特别的，标准的前端环境：

  • Node.js: v18 以上（硬性要求）
  • npm: 随 Node 安装

2.2 必需账号

我们需要两个服务的“入场券”：

  1. 智谱 AI (GLM): 我们的新“心脏”。
    • 注册地址
    • 费用：按量付费，新用户通常有赠送额度。
  2. Composio: 我们的“机械臂”（负责调用外部工具）。
    • 注册地址
    • 费用：免费版每月 20K 调用，个人开发足够了。

4. 效果验证：它能干什么？

启动后，你面对的是一个看似普通的聊天窗口，但背后已经是 GLM-4.7 + 500种工具 的完全体了。

4.1 基础测试：身份验证

问它：“你是谁？”

如果它回答“我是 Claude...”，别担心，这是因为系统提示词里写死了。

关键看后台日志，如果请求发往了 open.bigmodel.cn，说明替换成功！

4.2 进阶测试：联网搜索 (WebSearch)

输入：“帮我搜索一下 2025 年最新的 AI 技术趋势。”

幕后流程：

  1. GLM 判断需要联网。
  2. 调用 Composio 的 WebSearch 工具。
  3. Composio 抓取 Google/Bing 结果。
  4. GLM 总结结果并返回给你。

4.3 高级测试：本地文件操作 (File Edit)

输入：“在当前目录创建一个 hello_glm.py，内容是打印 1 到 100 的斐波那契数列。”

幕后流程：

  1. GLM 生成 Python 代码。
  2. 调用 Write 工具将代码写入你本地硬盘。

6. 避坑指南 (FAQ)

在实战中，你可能会遇到这几个坑，我都替你踩过了：

  • Q: 启动报 EADDRINUSE: 3001？
    • A: 上次关闭没关干净。终端运行 lsof -ti:3001 | xargs kill -9 杀掉进程。
  • Q: 智谱 API 报 401 错误？
    • A: 检查 .env 里的 Key 是不是智谱的（格式有点号），别填成了 Anthropic 的 Key。
  • Q: 为什么 Gmail 工具用不了？
    • A: 第一次使用敏感工具（GitHub/Gmail等），Composio 会在浏览器弹出一个 OAuth 授权窗口，你需要登录并授权。
